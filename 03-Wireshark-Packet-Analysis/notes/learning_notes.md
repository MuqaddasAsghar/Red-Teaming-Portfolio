# Learning Notes – Project 03

## Project Title

Wireshark Packet Analysis

---

# Objective

The objective of this project was to understand how network communication works by capturing and analyzing live packets using Wireshark.

Instead of only learning networking concepts theoretically, this project provided practical experience in inspecting real network traffic generated by different protocols.

---

# What I Learned

## Wireshark

- Installed and verified Wireshark.
- Started live packet capture.
- Selected the appropriate network interface.
- Used display filters to inspect specific protocols.

---

## TCP

I learned that TCP is a connection-oriented protocol used for reliable communication between systems.

During this project I observed:

- TCP packets
- Source and destination ports
- HTTPS communication over port 443

---

## DNS

Using the `nslookup` command, I generated DNS traffic and observed:

- DNS queries
- DNS responses
- Name resolution process

This helped me understand how domain names are translated into IP addresses.

---

## ARP

ARP traffic demonstrated how devices discover the MAC address of another device on the local network.

I observed:

- ARP Request
- ARP Reply

This clarified the relationship between IP addresses and MAC addresses.

---

## ICMPv6

I analyzed ICMPv6 packets generated by the system.

I observed Neighbor Discovery traffic, which is used in IPv6 networks for address resolution and network communication.

---

## HTTPS / TLS

Using:

```bash
curl https://example.com
```

I generated encrypted HTTPS traffic.

I observed:

- TLS handshake
- Server Hello
- Application Data

This demonstrated how encrypted communication occurs over HTTPS.

---

## Packet Filtering

I practiced using multiple Wireshark display filters including:

- tcp
- dns
- arp
- icmpv6
- tls

This improved my ability to isolate specific network protocols for analysis.

# Challenges Faced

Initially, ICMP packets did not appear as expected.

After troubleshooting, I successfully analyzed ICMPv6 traffic, which demonstrated IPv6 Neighbor Discovery messages generated by the system.

This reinforced the importance of understanding differences between IPv4 and IPv6 traffic.
