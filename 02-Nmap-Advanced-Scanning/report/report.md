# Project Report

# Project Title

Nmap Advanced Scanning

---

## Objective

The objective of this project was to explore advanced Nmap scanning techniques for host discovery, port scanning, service enumeration, UDP scanning, timing templates, Nmap Scripting Engine (NSE), and saving scan results in a controlled lab environment.

---

## Environment

- Operating System: Kali Linux 2025.1c
- Tool: Nmap 7.95
- Platform: Oracle VirtualBox
- Target: Localhost (127.0.0.1)

---

# Task 1 – Scan Without Host Discovery

## Command

```bash
nmap -Pn 127.0.0.1
```

### Observation

- Host was scanned without performing host discovery.
- Port 22 (SSH) was open.
- Remaining scanned TCP ports were closed.

### Conclusion

The `-Pn` option allows scanning a target even when host discovery is skipped.

---

# Task 2 – Scan a Specific Port

## Command

```bash
nmap -p 22 127.0.0.1
```

### Observation

- Port 22 was open.
- SSH service was detected.

### Conclusion

The `-p` option is useful for scanning only selected ports instead of performing a full scan.

---

# Task 3 – Scan Multiple Ports

## Command

```bash
nmap -p 22,80,443 127.0.0.1
```

### Observation

- Port 22: Open (SSH)
- Port 80: Closed (HTTP)
- Port 443: Closed (HTTPS)

### Conclusion

Scanning multiple selected ports provides faster results when only specific services need to be verified.

---

# Task 4 – Fast Scan

## Command

```bash
nmap -F 127.0.0.1
```

### Observation

- Port 22 was open.
- The scan checked the top 100 most common TCP ports.
- Scan completed quickly.

### Conclusion

The Fast Scan option reduces scanning time by checking fewer commonly used ports.

---

# Task 5 – Aggressive Timing Template

## Command

```bash
nmap -T4 127.0.0.1
```

### Observation

- Host responded successfully.
- SSH service remained accessible.
- Scan completed using the Aggressive timing template.

### Conclusion

The `-T4` timing template increases scanning speed and is commonly used during authorized penetration testing.

---

# Task 6 – UDP Scan

## Command

```bash
sudo nmap -sU 127.0.0.1
```

### Observation

- No open UDP services were detected.
- All scanned UDP ports were reported as closed.

### Conclusion

UDP scanning is useful for identifying UDP-based network services even though none were running in this lab.

---

# Task 7 – Default NSE Script Scan

## Command

```bash
nmap --script default 127.0.0.1
```

### Observation

- SSH service detected.
- SSH host keys (ECDSA and ED25519) were identified.

### Conclusion

The Nmap Scripting Engine (NSE) provides additional service information beyond standard port scanning.

---

# Task 8 – Save Scan Results

## Command

```bash
nmap -oN scan_results.txt 127.0.0.1
```

### Observation

- Scan results were successfully saved in a text file.

### Conclusion

Saving scan output improves documentation and simplifies future analysis.

---

# Task 9 – Verify Saved Results

## Command

```bash
cat scan_results.txt
```

### Observation

- The saved file accurately contained the scan results generated by Nmap.

### Conclusion

The saved output can be used for reporting, documentation, and later analysis.

---

# Overall Learning Outcomes

During this project, the following concepts were successfully practiced:

- Host Discovery Control (`-Pn`)
- Targeted Port Scanning (`-p`)
- Multiple Port Scanning
- Fast Scanning (`-F`)
- Timing Templates (`-T4`)
- UDP Scanning (`-sU`)
- Nmap Scripting Engine (NSE)
- Saving Scan Results (`-oN`)
- Viewing Saved Reports (`cat`)

---

# Final Conclusion

This project provided practical experience with advanced Nmap scanning techniques commonly used during the reconnaissance phase of authorized penetration testing and Red Teaming engagements. It demonstrated how different scanning options improve flexibility, efficiency, service enumeration, and documentation while operating within a controlled lab environment.
