# Learning Notes – Project 04

## What I Learned

During this project, I learned how passive information gathering is performed using common Linux networking tools. Instead of interacting directly with a target system, these tools collect publicly available information that helps understand a target's infrastructure.

---

## WHOIS

- Retrieves domain registration information.
- Shows registrar details.
- Displays creation and expiration dates.
- Lists authoritative name servers.

---

## DIG

- Queries DNS records.
- Retrieves A, AAAA, NS, MX and TXT records.
- Helps understand DNS configuration.

---

## HOST

- Resolves domain names into IP addresses.
- Displays IPv4 and IPv6 addresses.
- Can also display mail server information.

---

## NSLOOKUP

- Verifies DNS resolution.
- Queries DNS servers directly.
- Useful for troubleshooting DNS issues.

---

## CURL

- Retrieves HTTP response headers.
- Helps identify web server information.
- Shows security-related HTTP headers.

---

## TRACEROUTE

- Displays the path packets take across the network.
- Identifies intermediate network hops.
- Some routers may not respond due to firewall or security policies.

---

## Practical Outcome

This project improved my understanding of passive reconnaissance techniques used during the information gathering phase of penetration testing and Red Team engagements.

I also improved my documentation skills by recording commands, observations, screenshots, and project reports in a structured format.
